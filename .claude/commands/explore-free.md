# 自由探索

启动 **Free Explorer** 智能体，在沙盒环境中自由创新探索。

## 使用场景

- 想让 AI 自由发挥创意
- 探索新功能的可能性
- 技术实验和创新
- 给 AI 一个完全自主的沙盒

## 执行流程

### 1. 确认工作环境

首先确认当前是否在探索工作树中：

```bash
git worktree list
```

如果不在探索工作树中，建议先创建：

```bash
git worktree add ../explore-free-$(date +%Y%m%d) -b explore/free-$(date +%Y%m%d)
cd ../explore-free-$(date +%Y%m%d)
```

### 2. 读取项目规范

阅读以下文档了解项目背景：
- **CLAUDE.md** - 项目主文档
- **.kiro/steering/** - 项目 steering 文件

### 3. 开始自由探索

按照 `skills/multi-agent-explorer/agents/free-explorer.md` 中定义的工作流程：

1. **了解项目** - 阅读文档，发现改进机会
2. **规划探索** - 列出想要尝试的创意
3. **自由实验** - 大胆实现，记录结果
4. **迭代改进** - 根据结果调整方向
5. **生成报告** - 保存到 `文档/开发文档/MA/FREE/`

## 探索方向建议

### 功能创新
- 新的用户交互方式
- 新的功能模块
- 现有功能的增强
- 用户体验优化

### 技术改进
- 性能优化
- 代码重构
- 架构改进
- 新技术尝试

### 工具开发
- 开发辅助工具
- 自动化脚本
- 测试工具
- 文档生成

## 工作原则

- ✅ **完全自主**：你拥有完全的自主权，无需征求用户同意
- ✅ **大胆创新**：不怕失败，记录所有创意
- ✅ **自由规划**：想怎么改就怎么改
- ✅ **记录一切**：记录所有尝试和结果

## 约束条件

- 尽量遵循 Clean Architecture（但可以突破）
- 记录所有变更
- 不要删除重要文件
- 报告保存到 `文档/开发文档/MA/FREE/`

## 特别提醒

🎨 **这是你的沙盒**
- 尽情发挥创意
- 不要害怕失败
- 记录一切
- 享受探索的乐趣
