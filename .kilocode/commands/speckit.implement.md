---
description: 实施执行 - 执行实施计划，处理并执行 FD 功能文档中定义的所有任务
---

**说明：** 本命令用于执行实施计划，按照 FD 功能文档中定义的任务顺序和依赖关系，逐步完成所有实施任务。会先检查检查清单的完成状态，然后按阶段执行任务。

## 用户输入

```text
$ARGUMENTS
```

在继续之前，**必须**考虑用户输入（如果非空）。

## 执行流程

1. **查找文档**: 
   - 定位 FD 功能文档：`文档/开发文档/FD/FD-xxxxx-功能名称.md`
   - 定位 TDD 技术设计：`文档/开发文档/TDD/TDD-xxxxx-功能名称.md`
   - 定位 PRD 需求文档：`文档/开发文档/PRD/PRD-xxxxx-功能名称.md`

2. **检查检查清单状态**（如存在检查清单）：
   - 扫描所有检查清单文件
   - 对于每个检查清单，统计：
     - 总项目数：所有匹配 `- [ ]` 或 `- [X]` 或 `- [x]` 的行
     - 已完成项目：匹配 `- [X]` 或 `- [x]` 的行
     - 未完成项目：匹配 `- [ ]` 的行
   - 创建状态表：

     ```text
     | 检查清单 | 总数 | 已完成 | 未完成 | 状态 |
     |----------|------|--------|--------|------|
     | ux.md    | 12   | 12     | 0      | ✓ 通过 |
     | test.md  | 8    | 5      | 3      | ✗ 失败 |
     ```

   - **如果任何检查清单未完成**：
     - 显示带有未完成项目数的表格
     - **停止**并询问："部分检查清单未完成。是否仍要继续实施？（是/否）"
     - 等待用户响应后再继续

   - **如果所有检查清单都完成**：
     - 显示所有检查清单通过的表格
     - 自动继续下一步

3. **加载并分析实施上下文**：
   - **必需**: 读取 FD 获取完整任务列表和执行计划
   - **必需**: 读取 TDD 获取技术栈、架构和文件结构
   - **如存在**: 读取 data-model.md 获取实体和关系
   - **如存在**: 读取 contracts/ 获取 API 规格和测试要求
   - **如存在**: 读取 research.md 获取技术决策和约束

4. **项目设置验证**：
   - **必需**: 根据实际项目设置创建/验证忽略文件

   **检测与创建逻辑**：
   - 检查是否为 git 仓库 → 创建/验证 .gitignore
   - 检查是否存在 Dockerfile → 创建/验证 .dockerignore

   **按技术的常见模式**（来自 TDD 技术栈）：
   - **Kotlin/Android**: `build/`, `out/`, `.gradle/`, `.idea/`, `*.class`, `*.jar`, `*.iml`, `*.log`, `.env*`
   - **通用**: `.DS_Store`, `Thumbs.db`, `*.tmp`, `*.swp`, `.vscode/`, `.idea/`

5. **解析 FD 结构并提取**：
   - **任务阶段**: 设置、测试、核心、集成、收尾
   - **任务依赖**: 顺序 vs 并行执行规则
   - **任务详情**: ID、描述、文件路径、并行标记 [P]
   - **执行流程**: 顺序和依赖要求

6. **按任务计划执行实施**：
   - **逐阶段执行**: 完成每个阶段后再进入下一个
   - **尊重依赖**: 按顺序运行顺序任务，并行任务 [P] 可以一起运行
   - **遵循 TDD 方法**: 如需要编写测试，在相应实现任务之前执行测试任务
   - **基于文件的协调**: 影响相同文件的任务必须顺序运行
   - **验证检查点**: 在继续之前验证每个阶段完成

7. **实施执行规则**：
   - **先设置**: 初始化项目结构、依赖、配置
   - **测试先于代码**: 如需要为接口、实体和集成场景编写测试
   - **核心开发**: 实现模型、服务、用例、端点
   - **集成工作**: 数据库连接、中间件、日志、外部服务
   - **收尾和验证**: 单元测试、性能优化、文档

8. **进度跟踪和错误处理**：
   - 每个完成的任务后报告进度
   - 如果任何非并行任务失败则停止执行
   - 对于并行任务 [P]，继续成功的任务，报告失败的任务
   - 提供带有调试上下文的清晰错误消息
   - 如果实施无法继续则建议下一步
   - **重要** 对于已完成的任务，确保在 FD 文件中将任务标记为 [X]

9. **完成验证**：
   - 验证所有必需任务已完成
   - 检查实现的功能是否匹配原始规格
   - 验证测试通过且覆盖率满足要求
   - 确认实施遵循技术计划
   - 报告最终状态和已完成工作的摘要

注意：此命令假设 FD 功能文档中存在完整的任务分解。如果任务不完整或缺失，建议先运行 `/speckit.tasks` 重新生成任务列表。
