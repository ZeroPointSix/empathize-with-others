## 第二类：项目长期文档
这类文档是项目的长期资产，记录整体规范、架构设计和面向不同角色的指南。

### 规范
    该类文档统一存放在我们的"/文档/长期文档"路径下面
    并且具体的命名看具体编写文档的命名规则
    同时不同的文档划分是不一样的,例如:
    ```
    文档
        开发文档
                SD 
                    SD-00001-代码规范
                AD
                    AD-00001-悬浮窗架构设计
                API
                    API-00001-AI分析接口
                UM
                  UM-00001-悬浮窗使用指南
    .....
    ```



### 架构与规范类
- **规范文档 (SD - Specification Document)**
  - 命名规则: `SD-xxxxx-yyyy`
  - 示例: `SD-00001-代码规范`
  - 用途: 技术规范、编码标准、接口约定、项目整体规范

- **架构文档 (AD - Architecture Document)**
  - 命名规则: `AD-xxxxx-yyyy`
  - 示例: `AD-00001-悬浮窗架构设计`
  - 用途: 技术架构、项目架构、系统架构
- **API 文档 (API - API Documentation)**
  - 命名规则: `API-xxxxx-yyyy`
  - 示例: `API-00001-AI分析接口`
  - 用途: 接口定义、请求/响应格式、错误码说明

### 面向用户类
- **用户手册 (UM - User Manual)**
  - 命名规则: `UM-xxxxx-yyyy`
  - 示例: `UM-00001-悬浮窗使用指南`
  - 用途: 面向最终用户的功能使用说明

---

### 文档编号说明
- `xxxxx`: 5位数字编号，从00001开始递增
- `yyyy`: 简短的功能或需求描述，使用中文
- 同一需求的不同类型文档使用相同的编号
- 开发过程文档的编号与具体需求绑定
- 项目长期文档的编号按主题独立编号
