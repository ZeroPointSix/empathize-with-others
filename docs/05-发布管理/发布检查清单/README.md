# 发布检查清单

本目录包含发布前需要执行的检查项和验证步骤，确保发布质量和稳定性。

## 检查清单分类

### 1. 代码质量检查

#### 代码审查
- [ ] 所有代码变更已完成代码审查
- [ ] 代码审查通过率 ≥ 95%
- [ ] 关键模块有资深工程师审查
- [ ] 代码风格符合团队规范

#### 静态分析
- [ ] 静态代码分析无严重问题
- [ ] 代码覆盖率 ≥ 80%
- [ ] 安全扫描无高危漏洞

#### 单元测试
- [ ] 所有新增功能有单元测试
- [ ] 单元测试通过率 ≥ 95%
- [ ] 关键业务逻辑测试覆盖完整

### 2. 功能验证检查

#### 核心功能测试
- [ ] AI分析功能正常工作
- [ ] 安全检查功能正常工作
- [ ] 悬浮窗功能正常工作
- [ ] 联系人管理功能正常工作

#### 兼容性测试
- [ ] 微信环境下功能正常
- [ ] 不同Android版本兼容性验证
- [ ] 不同屏幕尺寸适配验证

#### 性能测试
- [ ] 应用启动时间 ≤ 3秒
- [ ] 内存使用在合理范围内
- [ ] CPU使用率 ≤ 30%（空闲时）
- [ ] 电池消耗测试通过

### 3. 文档完整性检查

#### 技术文档
- [ ] API文档已更新
- [ ] 架构文档已同步
- [ ] 变更日志已编写

#### 用户文档
- [ ] 用户手册已更新
- [ ] 新功能说明已添加
- [ ] 已知问题已记录

### 4. 发布准备检查

#### 版本管理
- [ ] 版本号符合命名规范
- [ ] 版本信息已正确配置
- [ ] 构建号与版本号一致

#### 构建验证
- [ ] Debug构建成功
- [ ] Release构建成功
- [ ] APK签名正确
- [ ] 安装包大小合理

#### 环境准备
- [ ] 测试环境已准备就绪
- [ ] 生产环境已备份
- [ ] 回滚方案已准备

### 5. 发布执行检查

#### 发布流程
- [ ] 发布计划已通知相关人员
- [ ] 发布窗口已确定
- [ ] 发布执行人员已指定
- [ ] 发布监控已启用

#### 应急预案
- [ ] 紧急联系人列表已更新
- [ ] 回滚步骤已验证
- [ ] 应急响应流程已确认
