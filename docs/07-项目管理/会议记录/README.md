# 会议记录

本目录记录项目会议、讨论要点和决策内容。

## 会议类型

### 1. 每日站会
- **目的**: 同步进度，识别阻塞，快速决策
- **频率**: 每个工作日上午
- **时长**: 15-30分钟
- **参与者**: 全体团队成员

### 2. 每周例会
- **目的**: 回顾本周进展，规划下周任务
- **频率**: 每周五下午
- **时长**: 60-90分钟
- **参与者**: 项目经理、技术负责人、核心开发人员

### 3. 月度回顾会
- **目的**: 总结月度成果，制定下月计划
- **频率**: 每月最后一周
- **时长**: 2-4小时
- **参与者**: 全体团队成员、管理层

### 4. 专题会议
- **目的**: 特定问题讨论，技术方案评审
- **频率**: 按需召开
- **时长**: 根据议题确定
- **参与者**: 相关领域专家、决策人员

## 会议记录格式

### 基本信息
- **会议主题**: 会议的主要议题和目的
- **会议时间**: 开始和结束时间
- **会议地点**: 线上/线下具体地点
- **主持人**: 会议主持人姓名
- **记录人**: 会议记录人姓名
- **参与者**: 出席人员名单

### 会议议程
1. **开场介绍** - 主持人介绍会议目的和议程
2. **进度同步** - 各模块负责人汇报当前进展
3. **议题讨论** - 按议程逐项讨论
4. **决策制定** - 对重要事项做出明确决策
5. **行动项确定** - 明确后续行动和责任人
6. **总结收尾** - 主持人总结会议成果和下一步

### 讨论要点
- **主要观点**: 讨论中的关键观点和意见
- **技术方案**: 提出的技术解决方案和实现方案
- **风险分析**: 识别的风险点和影响评估
- **资源需求**: 确定需要的人力、时间和其他资源

### 决策内容
- **决策事项**: 具体的决策内容和结论
- **决策理由**: 支持决策的关键理由和依据
- **实施计划**: 决策的具体实施步骤和时间安排
- **责任分配**: 各项任务的具体负责人和截止时间

## 会议最佳实践

### 会前准备
- 提前发送会议议程和相关材料
- 明确会议目标和预期成果
- 准备必要的数据和报告

### 会中记录
- 准确记录关键讨论点和决策
- 及时澄清模糊或不明确的观点
- 确保行动项具体可执行

### 会后跟进
- 及时整理和分发会议纪要
- 跟踪行动项的执行情况
- 为下次会议准备必要的材料
