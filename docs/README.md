## 🔧 开发规范

### 代码规范

遵循 **SOLID、KISS、DRY、YAGNI** 原则，详见 [CLAUDE.md](../CLAUDE.md)

### 提交规范

- **feat**: 新功能
- **fix**: Bug 修复
- **docs**: 文档更新
- **test**: 测试相关
- **refactor**: 重构

### 文档更新规范

1. **每次重要更新都要更新文档**
2. **废弃的文档移动到 04-Backup**
3. **保持 README.md 最新**
4. **更新时间戳和版本号**
