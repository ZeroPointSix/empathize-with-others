## 使用说明

### 添加任务

在"待执行任务"部分添加新任务：

```markdown
- [ ] 任务描述
  - 详细说明：xxx
  - 验收标准：xxx
  - 优先级：高/中/低
```

### 任务状态标记

| 标记 | 含义 |
|------|------|
| `[ ]` | 待执行 |
| `[x]` | 已完成 |
| `[-]` | 已跳过 |
| `[!]` | 执行失败 |

### 任务类型关键词

AI 会根据关键词自动识别任务类型：

| 关键词 | 类型 | 处理方式 |
|--------|------|----------|
| 修复、fix、bug | Bug 修复 | 调用调试策略 |
| 实现、开发、feature | 功能开发 | 按架构规范实现 |
| 测试、test | 测试编写 | 调用测试技能 |
| 重构、refactor | 代码重构 | 调用重构技能 |
| 文档、doc | 文档编写 | 调用文档技能 |
| 分析、analyze | 代码分析 | 调用分析技能 |

### 启动任务永动机

```
/task-runner
```

### 暂停执行

发送消息：
```
暂停任务执行
```

### 恢复执行

1. 执行 `/pickup` 恢复上下文
2. 执行 `/task-runner` 继续

---

## 注意事项

⚠️ **重要提醒**：

1. AI 会按顺序执行任务，想要优先执行的任务放在前面
2. 可以随时编辑此文件添加新任务
3. AI 每次循环都会重新读取此文件
4. 上下文快满时 AI 会自动 handoff
5. 失败的任务会标记 `[!]`，不会无限重试
