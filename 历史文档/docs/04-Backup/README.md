## 📋 归档规则

### 何时归档文档？

1. **文档被新版本替代**
   - 例如: 初期总结被最终总结替代

2. **阶段性文档已完成使命**
   - 例如: "如何开始 Step1" 在 Step1 完成后

3. **设计方案被推翻**
   - 保留原始设计用于参考

4. **临时性文档过期**
   - 例如: 问题排查记录

### 不应归档的文档

- ✅ **核心架构文档** - 即使有更新也应保留在主目录
- ✅ **API 文档** - 应该更新而非归档
- ✅ **测试文档** - 应该持续维护
- ✅ **开发指南** - 应该保持最新

## 🔍 如何使用归档文档

### 查找历史信息

如果你需要了解:
- 项目初期的决策过程
- 某个功能的原始设计
- 已废弃方案的实现细节

可以查看此文件夹中的相关文档。

### 恢复文档

如果某个归档文档需要恢复（例如发现该设计更合理）:

1. 复制文档到对应的主目录
2. 更新文档内容和时间戳
3. 在主文档目录添加引用
4. 可选: 从 Backup 中删除
