# IMPL-00014: 编译错误快速修复指南

## 问题

编译失败：`java.lang.NullPointerException` 在 `:app:packageDebug` 任务中

## 快速修复（3 步）

### 步骤 1：删除 Gradle 缓存

**Windows CMD**：
```cmd
rmdir /s /q %USERPROFILE%\.gradle\caches
rmdir /s /q .gradle
rmdir /s /q build
rmdir /s /q app\build
```

**Windows PowerShell**：
```powershell
Remove-Item -Recurse -Force $env:USERPROFILE\.gradle\caches
Remove-Item -Recurse -Force .gradle
Remove-Item -Recurse -Force build
Remove-Item -Recurse -Force app\build
```

### 步骤 2：清理项目

```bash
./gradlew clean
```

### 步骤 3：重新编译

```bash
./gradlew assembleDebug
```

---

## 如果仍然失败

### 执行完整重建

```bash
./gradlew --refresh-dependencies
./gradlew clean
./gradlew assembleDebug --info
```

### 查看详细日志

```bash
./gradlew assembleDebug --debug 2>&1 | tee build.log
```

然后查看 `build.log` 文件找出具体问题。

---

## 预期结果

✅ 编译成功，生成 `app/build/outputs/apk/debug/app-debug.apk`
