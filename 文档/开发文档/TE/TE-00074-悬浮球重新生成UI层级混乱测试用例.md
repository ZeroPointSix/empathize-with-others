# TE-00074 悬浮球重新生成 UI 层级混乱测试用例

## 目标
验证悬浮球最小化状态下重新生成的结果能正确回到悬浮窗显示，并确保正常场景不回归。

## 前置条件
- 已授予悬浮窗权限
- 已配置可用的 AI 服务商与模型
- 设备已连接（MuMu 或真机均可）

## 测试用例

### 用例 1：APP 外最小化后重新生成
**步骤**
1. 在其他应用界面打开悬浮窗
2. 输入内容并点击【帮我分析】生成结果
3. 点击最小化，切回悬浮球
4. 从悬浮球展开悬浮窗后点击【重新生成】

**预期**
- 悬浮窗自动展开或保持可见
- 重新生成结果在悬浮窗内显示
- 悬浮球状态与通知提示符合当前请求状态

### 用例 2：悬浮球展开后重新生成
**步骤**
1. 点击悬浮球展开悬浮窗
2. 点击【重新生成】

**预期**
- 重新生成结果在悬浮窗内显示
- UI 层级不混乱，无遮挡/不可见情况

### 用例 3：APP 内重新生成
**步骤**
1. 在 APP 内打开悬浮窗
2. 输入内容并点击【帮我分析】生成结果
3. 点击【重新生成】

**预期**
- 结果正常刷新在悬浮窗内
- 悬浮窗与悬浮球状态一致

## 回归检查
- 悬浮球最小化/展开流程正常
- 生成/复制/通知流程无异常
